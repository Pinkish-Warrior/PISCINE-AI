One-time setup

The curriculum is plain HTML/CSS/JavaScript — no build tools, no package.json, nothing to compile. Pick the path that matches where you're working.

Option A Own machine

  1. Install a code editor — VS Code, verified with code --version.
  2. Install Git — Homebrew, git-scm.com, or your distro's package manager.
  3. Clone the exercises repo 
    git clone https://github.com/01-edu/public.git
cd public/subjects/AI.GO
code .
  4. Sign up for an AI assistant — every exercise README ends with a Prompt Example. claude.ai or chatgpt.com; an in-editor extension is optional but recommended.
  5. Install Node.js — nodejs.org, verified with node --version. Needed once, for the next step.
  6. Run your code — opening index.html directly works early on, but file:// can break fetch and ES modules later, so serve it: 
    npx serve .

Option B College machine

  1. Log in — username student, password student. VS Code, Git and Node are already installed.
  2. Verify the tools are there — quick check before you rely on them: 
    code --version
git --version
node --version
    If any fail, install the missing one following Option A's steps for that tool.
  3. Clone the exercises repo 
    git clone https://github.com/01-edu/public.git
cd public/subjects/AI.GO
code .
  4. Sign up for an AI assistant — same as Option A, claude.ai or chatgpt.com.
  5. Run your code 
    npx serve .
Shared machine: log out of personal accounts (AI assistant, Gitea) at the end of a session, and confirm whether your repo clone gets pushed somewhere or is wiped between sessions.

Verify your setup

  • code --version prints a version
  • git --version prints a version
  • public/subjects/AI.GO exists locally after cloning
  • AI assistant account created
  • npx serve . (or opening index.html) shows a page in the browser

Pace & study tips

Task instructions themselves live in 01-edu/public/subjects/AI.GO — that's the source of truth, not duplicated here.

Suggested pace

Relaxed — 1 exercise/day~1 month
Intensive — 2 exercises/day~2 weeks
Between phasesexplain, don't just pass

Study tips for beginners

  1. Read before you code — understand the goal before touching the editor.
  2. Break things on purpose — change a value, see what happens.
  3. Use browser dev tools — console.log everything, inspect the DOM live.
  4. Don't skip phase 2 — variables, conditionals, and arrays show up in every exercise after.
  5. Revisit earlier exercises once later ones make them click in hindsight.
